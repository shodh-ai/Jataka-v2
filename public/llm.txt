## What is Jataka?

Jataka is a Backend Firewall and Developer Experience platform for Salesforce. We provide three core capabilities:

1. **Governor Limit Profiling** - Execute Apex code in isolated Sandboxes with Production-like data volumes, parse Debug Logs, and block PRs that would breach limits in Production.

2. **Self-Healing UI Tests** - Vision AI that automatically heals Playwright/Selenium/Cypress tests when Salesforce UI changes break selectors.

3. **Blast Radius Prediction** - Neo4j dependency graph integration with Cursor IDE via MCP protocol. Know what breaks before you change.

## Technical Architecture

### System Components

1. **One-Backend** - Central orchestration layer
   - Manages job queues
   - Handles OAuth authentication
   - Routes tasks to appropriate engines
   - Coordinates all Jataka services

2. **Brum (LLM)** - AI reasoning engine
   - Debug log analysis
   - Limit attribution
   - Code review suggestions
   - Test generation

3. **Kamikaze Pods** - Isolated execution environments
   - Fresh Sandbox instances
   - Production-like data volumes
   - No contamination, no shared state
   - Real transaction execution

4. **Vision AI** - Computer vision for UI testing
   - Salesforce Lightning UI trained models
   - Visual element recognition
   - Position and context awareness
   - Automatic selector healing

5. **Neo4j Dependency Graph** - Code relationship mapping
   - Every Apex class, trigger, flow mapped
   - Integration dependencies tracked
   - Real-time queryable
   - Blast radius calculation

### Kamikaze Pods
Isolated Sandbox environments that execute Apex code with Production-like data volumes. We measure real SOQL queries, DML statements, and CPU milliseconds—not static analysis estimates.

### Vision AI
Computer vision models trained on Salesforce Lightning UI. Recognizes elements by visual appearance, position, and context—not brittle CSS selectors.

### Neo4j Dependency Graph
Complete dependency map of your Salesforce org. Every Apex class, trigger, flow, and integration is mapped and queryable in real-time.

### MCP Protocol Integration
Model Context Protocol integration with Cursor IDE. Ask questions about your codebase and get answers before writing code.

## How It Works - Step by Step

### Governor Limit Profiling Flow

1. **GitHub Webhook Trigger** - PR opened/updated fires webhook
2. **One-Backend Orchestration** - Code fetched and queued
3. **Kamikaze Pod Spin-Up** - Isolated Sandbox provisioned
4. **Real Transaction Execution** - Apex code runs against real-ish data
5. **Debug Log Parsing** - SOQL, DML, CPU, heap metrics extracted
6. **Threshold Comparison** - Measured vs Governor Limits
7. **PR Status & Report** - Pass/Fail posted with detailed attribution

### Self-Healing UI Test Flow

1. **Test Failure Detection** - Selenium/Playwright test fails
2. **Screenshot Capture** - Current page state captured
3. **Vision AI Analysis** - Element identified visually
4. **Selector Generation** - New selector created
5. **Test Re-run** - Automatic verification
6. **PR Update** - Selector fix committed

### Blast Radius Prediction Flow

1. **Code Change Identification** - Developer opens file in Cursor
2. **Graph Query** - Neo4j queried for dependencies
3. **Impact Analysis** - Downstream effects calculated
4. **Risk Assessment** - Critical paths highlighted
5. **Developer Notification** - Results shown in IDE
